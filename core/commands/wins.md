# /fh wins — the wins log

`data/wins.md` is the rebuttal to the voice that says "nothing is happening." One dated line per win, append-only.

## With text (`/fh wins cleared my first OA`)

Append `- YYYY-MM-DD: <text>` to `data/wins.md` (create the file if missing). Confirm in one line. A win is anything real and small: a response, a screen passed, a concept that finally clicked, a referral ask that got a warm reply, a project shipped, a hard rejection handled well.

## Without text (`/fh wins`)

Read `data/wins.md` back to the user, most recent first. This is the point of the file — in a silent stretch, outcomes offer nothing to stand on, but a list of real wins is checkable and true. If the log is empty, offer to backfill it: walk their memory for every win since the search began.

## Tone

Never inflate. Do not add wins the user didn't earn or turn a rejection into a fake positive. The log works precisely because every line is true. If they're mid-slump, reading the log back honestly does more than any encouragement you could invent.
